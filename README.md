**csv2xyd-NDMOutAnalyzer**, a graphical workflow for cleaning biodiversity occurrence datasets and exporting files compatible with NDM/vNDM.

NDMOutAnalyzer, a companion module for parsing, filtering, comparing, and visualizing technical output files generated by NDM/vNDM.

Together, these tools support a reproducible workflow from occurrence records to comparative endemism analysis, including XYD file generation, .out file parsing, consensus-area comparison, spatial overlap analysis, beta-diversity decomposition, Excel reporting, and publication-ready consensus mosaics.

The project integrates two complementary components:

1. **csv2xyd**, a graphical workflow for cleaning biodiversity occurrence datasets and exporting files compatible with **NDM/vNDM**.
2. **NDMOutAnalyzer**, a companion module for parsing, filtering, comparing, and visualizing technical output files generated by **NDM/vNDM**.

Together, these tools support a reproducible workflow from occurrence records to comparative endemism analysis, including XYD file generation, `.out` file parsing, consensus-area comparison, spatial overlap analysis, beta-diversity decomposition, Excel reporting, and publication-ready consensus mosaics.

---

## Why this tool?

Open biodiversity repositories such as GBIF, ALA, and speciesLink provide access to millions of occurrence records. However, transforming these records into specialized inputs for areas of endemism analysis often requires substantial manual work: data cleaning, column standardization, coordinate filtering, taxonomic checking, format conversion, and post-analysis tabulation.

This bottleneck is especially relevant for users of **NDM/vNDM**, where occurrence records must be transformed into specific spatial matrix formats and where output files can be difficult to compare across grid sizes, consensus parameters, taxonomic treatments, or score thresholds.

**csv2xyd-NDMOutAnalyzer** reduces this gap by providing a desktop graphical workflow that helps users move from raw biodiversity occurrence tables to structured and interpretable outputs for biogeographic analysis.

## Main features

### 1. Occurrence data preparation with csv2xyd

The `csv2xyd` module provides a GUI-based workflow for preparing biodiversity occurrence datasets.

Main functions include:

* Loading biodiversity occurrence tables in CSV format.
* Mapping columns from different data sources.
* Removing duplicated coordinate records.
* Filtering taxa and occurrence records.
* Detecting possible taxonomic or geographic inconsistencies.
* Exporting files compatible with NDM/vNDM workflows.
* Generating XYD and TNT-compatible output files.

This module is intended for users who need to transform occurrence records into formats required by downstream cladistic biogeography and endemism analyses.

---

### 2. NDM-vNDM output analysis

The `NDMOutAnalyzer` module reads one or multiple `.out` files generated by NDM or vNDM and converts them into structured tabular outputs.

Main functions include:

* Loading multiple `.out` files from independent analyses.
* Filtering endemic species by minimum and maximum endemicity score.
* Including or excluding supraspecific categories using the `GRP-` prefix.
* Extracting individual endemic areas.
* Extracting consensus areas.
* Recovering endemic species lists and scores.
* Summarizing species and genera by area or consensus.
* Exporting all results to a multi-sheet Excel workbook.

This is especially useful for comparing analyses performed under different grid resolutions, score thresholds, consensus parameters, or taxonomic treatments.

---

### 3. Geospatial consensus analysis

The geospatial module compares consensus areas using paired polygon and point files associated with NDM/vNDM-related workflows.

Expected paired input files:

```text
conX_N.csv
pointX_N.csv
```

Where:

```text
conX_N.csv    Polygon representation of consensus area X.
pointX_N.csv  Species occurrence points associated with consensus area X.
N             Number of individual areas contributing to the consensus.
```

Polygon files may contain either:

```text
WKT
```

or coordinate columns such as:

```text
lon, lat
```

or:

```text
x, y
```

The geospatial workflow calculates:

* Consensus area size.
* Number of species per consensus area.
* Polygon area.
* Pairwise spatial overlap.
* Jaccard area index.
* Shared species.
* Shared genera.
* Exclusive species and genera.
* Pairwise beta-diversity components.

---

### 4. Beta-diversity decomposition

For each pair of consensus areas, the software estimates taxonomic dissimilarity metrics, including:

* Jaccard taxonomic dissimilarity.
* Sørensen dissimilarity.
* Turnover component.
* Nestedness component.

These metrics help evaluate whether differences among endemic areas are mainly explained by species replacement or by species loss/gain between areas.

---

### 5. Consensus mosaic export

The mosaic module generates high-resolution multi-panel figures for visual comparison of consensus areas across grid resolutions or analytical treatments.

Main functions include:

* Automatic detection of consensus polygon files.
* Optional loading of external basemaps in `.shp` or `.gpkg` format.
* Consistent geographic extent across all panels.
* Export of high-resolution PNG mosaics.
* Figure outputs suitable for reports, presentations, and manuscript preparation.

## Installation

### Install with pip

Clone the repository:

```bash
git clone https://github.com/NCBC-Lab/csv2xyd-NDMOutAnalyzer.git
cd csv2xyd-NDMOutAnalyzer
```

Install dependencies:

```bash
pip install pandas numpy dask fuzzywuzzy folium geopandas shapely XlsxWriter matplotlib
```

Optional fuzzy-matching speedup:

```bash
pip install python-Levenshtein
```

On Ubuntu or Debian-based Linux systems, install Tkinter if needed:

```bash
sudo apt install python3-tk
```

Run the main workflow:

```bash
python csv2xyd.py
```

Or run the NDM/vNDM output analyzer directly:

```bash
python NDMoutAnalyzer.py
```

## Basic workflow

### Step 1. Prepare occurrence records

1. Launch `csv2xyd.py`.
2. Load a biodiversity occurrence table in CSV format.
3. Map the required taxonomic and coordinate columns.
4. Apply cleaning and filtering options.
5. Export files compatible with NDM/vNDM.

---

### Step 2. Run NDM or vNDM

Run NDM or vNDM externally using the files generated by `csv2xyd`.

The resulting `.out` files can then be processed with `NDMOutAnalyzer`.

---

### Step 3. Analyze NDM-vNDM output files

1. Open the NDM/vNDM output analyzer.
2. Load one or more `.out` files.
3. Define minimum and maximum endemicity score thresholds.
4. Choose whether to include or exclude `GRP-` supraspecific categories.
5. Export the parsed results to Excel.

The exported workbook includes:

* General analysis statistics.
* Individual endemic areas.
* Consensus endemic areas.
* Species-level summaries.
* Genus-level summaries.
* Endemic species information.
* File-level statistics.

---

### Step 4. Run geospatial consensus analysis

1. Select a root folder containing grid-size or treatment subfolders.
2. Each subfolder should contain paired `conX_N.csv` and `pointX_N.csv` files.
3. Run the geospatial comparison.
4. Export the results to Excel.

The output includes:

* Consensus summary.
* Spatial overlap statistics.
* Jaccard area index.
* Shared species and genera.
* Exclusive species and genera.
* Pairwise beta-diversity metrics.

---

### Step 5. Export consensus mosaics

1. Select the root folder containing consensus polygon files.
2. Optionally select a basemap file in `.shp` or `.gpkg` format.
3. Export the consensus mosaic as a high-resolution PNG image.
